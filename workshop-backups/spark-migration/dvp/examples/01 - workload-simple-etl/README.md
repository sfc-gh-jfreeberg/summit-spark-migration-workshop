# Example 01: Simple ETL Workload

A small PySpark ETL workload with 3 pipelines, designed to exercise the core DVP skills. The output was generated using the real SMA tool.

## Pipelines

| Pipeline | File | Entry Point | Description |
|----------|------|-------------|-------------|
| Customer Ingestion | `pipelines/customer_ingestion.py` | `if __name__ == "__main__":` | Reads customer CSV from S3, transforms, writes to table |
| Order Processing | `pipelines/order_processing.py` | `def main():` | Reads from orders table, joins with customers, writes summary |
| Daily Report | `jobs/daily_report.py` | `if __name__ == "__main__":` | Aggregates data and writes a daily report table |

## Non-Pipeline Files

| File | Type | Description |
|------|------|-------------|
| `utils/spark_helpers.py` | Utility | Shared helper functions (session creation, logging) |
| `utils/transformations.py` | Utility | Reusable DataFrame transformations |
| `config/settings.py` | Config | Configuration constants |

## I/O Summary

| Pipeline | Inputs | Outputs |
|----------|--------|---------|
| Customer Ingestion | `s3://data-lake/raw/customers.csv` | `db.schema.dim_customers` |
| Order Processing | `db.schema.orders`, `db.schema.dim_customers` | `db.schema.order_summary` |
| Daily Report | `db.schema.order_summary` | `db.schema.daily_report` |

## Usage

```
<input>  = examples/01 - workload-simple-etl/input
<output> = examples/01 - workload-simple-etl/output/Conversion-2-12-2026T13 8  (auto-resolved from .snowma)
```

Point the orchestrator at `<input>` and let it resolve `<output>` from the `.snowma` file.
