# Example 02: ECommerceDataPipeline

A single-file e-commerce data pipeline that reads 5 CSV inputs, performs multiple transformations (aggregation, joins, window functions, currency conversion), and produces 5 output tables. Output was generated using the real SMA tool.

## Pipeline

| Pipeline | File | Entry Point | Description |
|----------|------|-------------|-------------|
| ECommerceDataPipeline | `workload.py` | `if __name__ == "__main__":` | Full ETL pipeline: reads CSVs, transforms, writes to tables |

## Inputs (5 CSV files)

| Input | Description |
|-------|-------------|
| `data/raw_transactions.csv` | Transaction records (id, date, amount, customer, product) |
| `data/customer_master.csv` | Customer master data (id, name, country) |
| `data/product_catalog.csv` | Product catalog (id, name, category) |
| `data/returns_data.csv` | Return records (transaction_id, return_amount) |
| `data/exchange_rates.csv` | Exchange rates by date and currency |

## Outputs (5 tables)

| Output Table | Transformation | Description |
|-------------|---------------|-------------|
| `DAILY_SALES_SUMMARY` | GroupBy + Agg | Daily revenue, transaction count, returned revenue |
| `CUSTOMER_CLV` | Join + Agg | Customer lifetime value (total spend, net spend) |
| `TOP_CATEGORIES` | GroupBy + Window (RANK) | Top 5 product categories by sales count |
| `FOREIGN_TRANSACTIONS` | Join + Enrichment | Transactions converted to USD using exchange rates |
| `UNMATCHED_TRANSACTIONS` | Filter + When/Otherwise | Error log: high-value or missing-customer transactions |

## Expected Output Data

This example includes **expected output data** in `data/expected_output/` for each of the 5 output tables. These can be used by `dvp-test-validation-generator` for assertion comparison.

| File | Rows |
|------|------|

## Key Characteristics

This example is useful for testing DVP skills because it features:

- **Single-file pipeline** -- all logic in one `workload.py` (tests single-file detection)
- **Multiple I/O patterns** -- CSV reads via `format("csv").load(...)`, table writes via `saveAsTable()`
- **Embedded SQL** -- `spark.sql(...)` calls with `SELECT`, `CREATE TABLE`, `DROP TABLE`
- **Window functions** -- `RANK() OVER (ORDER BY ...)` for top categories
- **Real EWIs** -- 20 issues across 7 different EWI codes for `dvp-ewi-*` skills
- **Expected output data** -- pre-computed results for validation comparison
- **SQL conversion issues** -- `!!!RESOLVE EWI!!!` markers in SQL strings
